## 🎯 项目概述	

本仓库是一家**跨境电商公司**（宠物行业，亚马逊平台）的数据仓库建设文档中心，包含数据仓库规划、总线矩阵构建、数据域定义等核心交付物。	

### 核心目标	

- 构建完整的 **Kimball 总线矩阵（Bus Matrix）**	
- 定义 **16 个数据域**的业务过程与维度模型	
- 建立标准化的数据开发规范体系

## 📊 数据域概览	

本项目定义了 **16 个数据域**，按优先级分级：	

### P0 级 - 核心域（最高优先级）	

| 数据域 | 编码 | 核心业务 |	
|--------|------|----------|	
| **交易域** | `trd` | 订单、支付、结算 |	
| **运营域** | `op` | 账号、店铺、链接、流量 |	

### P1 级 - 重要域	

| 数据域 | 编码 | 核心业务 |	
|--------|------|----------|	
| **库存域** | `inv` | 入库、出库、库龄 |	
| **履约域** | `fl` | 物流、FBA、退换货 |	
| **促销域** | `prm` | 促销活动、策略、费用 |	
| **广告域** | `ads` | 广告投放、效果、流量 |	

### P2 级 - 一般域	

| 数据域 | 编码 | 核心业务 |	
|--------|------|----------|	
| **研发域** | `rd` | 产品设计、迭代、BOM |	
| **供采域** | `sup` | 采购计划、订单、供应商 |	
| **生产域** | `mfg` | 产能、排产、工单 |	
| **品控域** | `qc` | 质检、缺陷追溯 |	

### P3 级 - 辅助域	

| 数据域 | 编码 | 核心业务 |	
|--------|------|----------|	
| **市场域** | `mkt` | 市场调研、竞品分析 |	
| **客服域** | `cs` | 客诉、工单 |	
| **财务域** | `fin` | 应收应付、利润核算 |	
| **客户域** | `cst` | 客户画像、复购 |	
| **政策域** | `pol` | 政策解析、规则追踪 |	
| **算法域** | `ma` | 算法元数据、特征 |

## 📋 命名规范	

遵循数据开发规范，采用统一的命名约定：	

### 表命名格式	

```
{层级}_{域缩写}_{业务对象}_{粒度}_{刷新周期}
```

**示例**：	
- `ods_trd_amz_order_main_di` - ODS层交易域亚马逊订单主表（日增量）	
- `dwd_inv_stock_snapshot_dd` - DWD层库存域库存快照表（日全量）	
- `dws_ads_campaign_effect_md` - DWS层广告域活动效果表（月度）	

### 层级说明	

| 层级 | 说明 |	
|------|------|	
| `ods` | 操作数据层 - 原始数据 |	
| `dwd` | 明细数据层 - 清洗后明细 |	
| `dws` | 汇总数据层 - 轻度汇总 |	
| `dim` | 维度层 - 维度表 |	
| `ads` | 应用数据层 - 应用宽表 |
